CNN:
1. Local connectivity:
1
•Restrict the output units of a layer to only be aﬀected by input units
spatially close (while in fully connected NN the output of each layer is
connected to every single input unit).
•Result in a reduction of the dimensionality of the weights β(replacing the
matrix multiplication used in fuly connected layers by a convolution with
a kernel that is smaller than the dimensionality of the input features).
•Intuition: local connectivity utilizes the strong 2D local structure in im-
ages assuming interactions between pixels that are spatially close ought
to be more important than features that are further apart.
2. Parameter sharing:
•Based on the assumption we want an equivariance to translation in images.
If an important feature is detected in the image and then the input shifted,
the feature should be shifted by the same amount in the output.
•Parameter sharing is implemented by using the same set of parameters
(e.g. same kernel function) at diﬀerent parts of the image.
•Reduce the memory storage required.
3. Pooling:
•Replace the output of the layer by summary statics.
•Make the invariant to small translations of the input.
The only diﬀerence between convolutional and fully connected NN in the full
data DL algorithm is which type of layer is utilized (in step one).
